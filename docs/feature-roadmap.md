================================================================================
          CRANEAPP MESSENGER — DEVELOPMENT ROADMAP & MILESTONES
================================================================================
Версия: 1.0.0
Стратегия: "Infrastructure First, UI Second, Polish Last"
Модель разработки: 7 Спринтов до Production Ready

--------------------------------------------------------------------------------
СПРИНТ 1: CORE INFRASTRUCTURE & AUTH (ФУНДАМЕНТ)
--------------------------------------------------------------------------------
Цель: Создать работающую систему регистрации и шлюз для микросервисов.

Задачи:
1.1. Настройка API Gateway (Express + Proxy): Маршрутизация запросов.
1.2. База данных: Инициализация PostgreSQL (Users, Sessions) и Redis.
1.3. Auth Service:
     - Реализация логики OTP (генерация, хранение в Redis, верификация).
     - Генерация JWT Access/Refresh токенов.
     - Middleware для проверки авторизации.
1.4. Базовый UI:
     - Экран ввода телефона.
     - Экран ввода кода.
     - Сохранение токенов в LocalStorage/Cookies.

Результат: Пользователь может войти в систему и получить токен.

--------------------------------------------------------------------------------
СПРИНТ 2: USER PROFILES & CONTACTS (СОЦИАЛЬНЫЙ ГРАФ)
--------------------------------------------------------------------------------
Цель: Управление личностью пользователя и его связями.

Задачи:
2.1. User Service:
     - Создание/Редактирование профиля (Имя, Username, Bio).
     - Загрузка аватара (Интеграция с Media Service).
2.2. Contact System:
     - Поиск пользователей по Username.
     - Добавление в контакты, синхронизация телефонной книги.
     - Глобальный поиск (Global Search).
2.3. Статусы:
     - Логика Online/Offline/Last Seen (Redis-based).

Результат: Пользователи видят друг друга и могут менять свои данные.

--------------------------------------------------------------------------------
СПРИНТ 3: REAL-TIME MESSAGING ENGINE (ЯДРО МЕССЕНДЖЕРА)
--------------------------------------------------------------------------------
Цель: Запуск обмена текстовыми сообщениями в реальном времени.

Задачи:
3.1. Socket.io Setup: Настройка WebSocket сервера и распределение по комнатам (Room per Chat).
3.2. Message Service:
     - Отправка сообщения (msg:send).
     - Сохранение в DB (PostgreSQL).
     - Мгновенная рассылка участникам.
3.3. UI Чат:
     - Message Bubbles (стили из UI-Guidelines).
     - Автоскролл вниз.
     - Индикаторы доставки (одна галочка / две галочки).
3.4. Список чатов:
     - Сортировка по времени последнего сообщения.
     - Счетчик непрочитанных (Unread badge).

Результат: Полноценная текстовая переписка 1-на-1.

--------------------------------------------------------------------------------
СПРИНТ 4: GROUPS, CHANNELS & RICH CONTENT (МАСШТАБИРОВАНИЕ)
--------------------------------------------------------------------------------
Цель: Групповые взаимодействия и расширенный контент.

Задачи:
4.1. Chat Service:
     - Создание групп (Group Chats).
     - Управление участниками (Add/Remove/Promote to Admin).
     - Создание публичных каналов (Channels).
4.2. Rich Content:
     - Редактирование и удаление сообщений (Edit/Delete).
     - Ответы (Replies) и Пересылка (Forwarding).
     - Реакции на сообщения (Emoji Reactions).
4.3. Media Service (Phase 2):
     - Отправка фото и видео.
     - Генерация превью (Thumbnails) на лету.

Результат: Функционал на уровне полноценных групп Telegram.

--------------------------------------------------------------------------------
СПРИНТ 5: VOICE & VIDEO CALLS (WEBRTC INTEGRATION)
--------------------------------------------------------------------------------
Цель: Внедрение системы звонков высокой четкости.

Задачи:
5.1. Call Service (Signalling):
     - Обработка событий offer/answer/candidate.
     - Управление очередью звонков.
5.2. WebRTC Client Logic:
     - Захват медиа-потоков (Mic/Camera).
     - Настройка STUN/TURN серверов для обхода NAT.
5.3. Call UI (по промпту):
     - Экран входящего/исходящего вызова.
     - Панель управления звонком (Mute, Flip, End).
     - Групповые голосовые чаты.

Результат: Возможность звонить друг другу внутри Craneapp.

--------------------------------------------------------------------------------
СПРИНТ 6: ADVANCED FEATURES & SECURITY (ПРОФЕССИОНАЛЬНЫЙ СЛОЙ)
--------------------------------------------------------------------------------
Цель: Безопасность и дополнительные удобства.

Задачи:
6.1. Encryption: Внедрение End-to-End Encryption для "Секретных чатов".
6.2. Search: Полнотекстовый поиск по истории сообщений (PostgreSQL GIN Index).
6.3. Stickers & GIFs: Интеграция с API (например, Tenor) + собственные паки стикеров.
6.4. Settings: Настройки уведомлений, приватности и темы (Light/Dark).

Результат: Завершенный продукт со всеми "фишками" конкурентов.

--------------------------------------------------------------------------------
СПРИНТ 7: DEPLOYMENT, OPTIMIZATION & RAILWAY (РЕЛИЗ)
--------------------------------------------------------------------------------
Цель: Стабильный запуск в облаке.

Задачи:
7.1. Infrastructure:
     - Написание Dockerfile для каждого сервиса.
     - Настройка Docker Compose для локального запуска.
7.2. Railway Cloud:
     - Деплой микросервисов на Railway.
     - Настройка Managed PostgreSQL и Redis.
     - Настройка Health Checks и авто-рестартов.
7.3. Optimization:
     - Кэширование запросов в Redis.
     - Сжатие статики (Gzip/Brotli).
     - Минификация JS/CSS.

Результат: Публичная ссылка на работающий мессенджер.
